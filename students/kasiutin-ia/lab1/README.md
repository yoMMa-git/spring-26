# Реализация алгоритма
Алгоритм ID3 реализует следующую логику ветвления в зависимости от типа выбранной переменной:
- Числовые признаки: бинарное разбиение по порогу; перебираются середины между соседними уникальными значениями, выбирается порог с максимальным приростом.
- Бинарные признаки: логика повторяет работу с числовыми признаками.
- Категориальные признаки: многозначное разбиение по значениям признака.

Для ветвления используется критерий Джини. Его формула в общем случае: 

$$G(p) = 1 - Σp_i^2$$

Однако для бинарного случая можно упростить:

$$G(p) = 2p(1-p),$$

где $p$ - вероятность класса 1.

# Обработка пропусков

- **При обучении:** при расчёте $Gain(f, U)$ объекты с пропуском по признаку $f$ не учитываются. Для каждой внутренней вершины сохраняются оценки вероятностей ветвей: $$q_{vk} = |U_k| / |U|$$ (по объектам с известным значением $f$). В листьях хранится распределение $P(y|U)$ по попавшей в лист подвыборке.
- **При классификации:** если значение признака в узле известно - переход в соответствующую дочернюю вершину; если пропущено - используется взвешенное распределение по всем дочерним:  
  $P(y|x,v) = Σₖ q_vk · P(y|x, S_v(k))$

  Ответ: $a(x) = argmax_y P(y|x, v0)$

---

Редукция дерева (REP)

Реализован Reduced Error Pruning по валидационной выборке: обход дерева снизу вверх; для каждой внутренней вершины проверяется замена поддерева на лист с мажоритарным классом (по объектам валидации, дошедшим до этого узла). Если точность на валидации не ухудшается, поддерево заменяется листом.

# Результаты экспериментов

В качестве готовой реализации бралась модель DecisionTreeClassifier из пакета Sklearn. Инициализация происходила следующим образом:
```
DecisionTreeClassifier(criterion="gini", random_state=42, min_samples_leaf=5, max_depth=20)
```
Все пропущенные значения заменялись медианой.


| Модель            | Accuracy | F1 (weighted) | Precision | Recall |Узлов | Листьев |
|-------------------|----------|----------------|-------|------|------|---------|
| ID3 до редукции   | 0.8096 | 0.7883 | 0.6345 | 0.3548 | 7570 | 5252   |
| ID3 после редукции     | 0.8285 | 0.8115  | 0.6345  | 0.3548 | 1758 | 1329  |
| Sklearn | 0.8106 | 0.8068  | 0.5863  |  0.5263 | 12349 | 6175 |

- **До и после редукции:** после REP дерево обычно становится меньше (меньше узлов и листьев); accuracy и F1 на тесте выросли, вероятно, за счет снижения переобучения.
- **Сравнение с sklearn:** эталонная реализация `DecisionTreeClassifier(criterion="gini")` на тех же данных (с заполнением пропусков медианой для sklearn) даёт сопоставимое качество; наша реализация с явной обработкой пропусков через `q_vk` позволяет не заполнять пропуски и использовать взвешенное прогнозирование в узлах.
